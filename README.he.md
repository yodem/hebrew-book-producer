# hebrew-book-producer — מסמך אדריכלות

פלאגין Claude Code להפקה מקצועית של ספרי עיון, אוטוביוגרפיה, פילוסופיה וטקסטים דתיים בעברית. הפלאגין מדמה את שרשרת הערך של הוצאת ספרים ישראלית — מהערכת כתב היד הראשונית ועד לקובץ המוכן לדפוס — באמצעות מערכת סוכנים אוטונומית הפועלת מקומית.

## התקנה — לפי פרויקט (מומלץ)

הפלאגין **אינו** נטען אוטומטית ב־Claude Code, ובד״כ אין סיבה לטעון אותו גלובלית — הוא רלוונטי רק בתיקיות שיש בהן כתב יד.

### דרך A — התקנה מ־GitHub, הפעלה לפי פרויקט

```bash
/plugin marketplace add yodem/hebrew-book-producer
/plugin install hebrew-book-producer@hebrew-book-producer
```

עכשיו, **בתוך תיקיית הפרויקט** של הספר:

```bash
cd /path/to/your/book-project
mkdir -p .claude
cat > .claude/settings.json <<'EOF'
{
  "enabledPlugins": {
    "hebrew-book-producer@hebrew-book-producer": true
  }
}
EOF
```

הפלאגין כעת פעיל **רק** כש־Claude Code מופעל מהתיקייה הזאת. הפעל מחדש את Claude Code מתוך התיקייה ובדוק עם `/help`.

### דרך B — התקנה מקומית לפיתוח

```bash
git clone https://github.com/yodem/hebrew-book-producer.git ~/dev/hebrew-book-producer
```

ובתיקיית הפרויקט:

```bash
cd /path/to/your/book-project
mkdir -p .claude
cat > .claude/settings.json <<'EOF'
{
  "enabledPlugins": {
    "hebrew-book-producer@local": true
  },
  "extraKnownMarketplaces": {
    "local": {
      "source": { "source": "local", "path": "/Users/YOU/dev" }
    }
  }
}
EOF
```

החלף את `/Users/YOU/dev` בתיקיית האם של הריפו ששכפלת.

### gitignore

```bash
echo ".claude/" >> .gitignore
```

### תלויות ריצה (אופציונליות)

```bash
# CandleKeep CLI — שכבת הידע (מדריך כתיבה + Hebrew Linguistic Reference)
curl -fsSL https://candlekeep.dev/install.sh | sh
ck auth login

# Python — לחילוץ טביעת אצבע סגנונית במסלול הכבד של /init-voice
pip install pdfplumber python-docx
```

Sefaria MCP מגיע עם הכלים של claude.ai; רק לאשר כשמבקשים.

## התחלה מהירה — משפט אחד

שמור את כתב היד כ־`manuscript.md` (או פרקים תחת `chapters/`). פתח את התיקייה ב־Claude Code וכתוב באופן חופשי, בעברית או באנגלית:

| מה אתה כותב | מה קורה |
|---|---|
| **תוכל להגיה את הספר שלי?** | זיהוי כתב היד, יצירת קבצי הפרויקט, 3 שאלות זריזות על הקול, הרצת הגהה |
| תוכל לערוך את הספר | עריכה ספרותית + לשונית |
| תקרא ותגיד לי מה אתה חושב | חוות דעת לקטור |
| תעמיד | בריף לעימוד |
| תכתוב לי פרק 3 | ה־book-writer מנסח פרק לפי ברייף |
| ספר חדש | יצירת פרויקט ריק |

סה״כ קלטים להגהת ספר קיים: **משפט אחד + אישור אחד + 3 תשובות על הקול**. לא צריך לכתוב YAML.

הראוטר ב־`CLAUDE.md` ממפה אוטומטית את הניסוח החופשי שלך לפקודה `/start <פעולה>`.

## ארבעת צמתי ההפקה

| שלב | תפקיד מקצועי | סוכן | אחריות |
|---|---|---|---|
| הערכה וייעוץ | עורך ראשי / לקטור | `lector` + `production-manager` | קביעת מסלול, ניהול לוחות זמנים |
| נרטיב ומבנה | עורך ספרותי | `literary-editor` | חידוד דמויות וטיעונים, ארגון פרקים |
| לשון וסגנון | עורך לשוני | `linguistic-editor` | האחדת משלב, קריאות, מניעת חזרות |
| ליטוש סופי | מגיה | `proofreader` | תיקון שגיאות, פיסוק, האחדת כתיב |
| עיצוב הטקסט | מעמד / גרפיקאי | `typesetting-agent` | טיפוגרפיה, שוליים, כותרות רצות |

ההגהה אינה ליניארית: לפחות שלושה סבבים — אחרי עריכה לשונית, אחרי עימוד, ולפני הדפסה סופית. מנהל ההפקה מתאם את הלולאות.

## תלויות ריצה — שכבת הידע

הפלאגין נשען על שני רכיבים אופציונליים. ללא אף אחד מהם הוא עדיין עובד במצב יחיד־סוכן.

1. **CandleKeep — שכבת הידע של המחבר.** מאחסן את הידע המאוצר של המחבר: מדריך הכתיבה (King/Zinsser/Penn/Shapiro — `cmok9h0m10ahik30zt8yt0lt2`), מדריך לבניית צוות סוכנים (`cmnudfue5003rmy0zlxt7ioa1`), מחברת תזה פרטית לספר זה (`book.yaml: thesis_notebook`), ומדריכי כתיבה נוספים (`craft_extras: [...]`). **לא משמש לאחסון מקורות דתיים קנוניים** — אלו עוברים דרך Sefaria.
2. **Sefaria — אימות מקורות.** כל ציטוט מחז"ל / תנ"ך / בבלי / ירושלמי / מדרש / רמב"ם / שולחן ערוך מאומת מול Sefaria דרך כלי MCP (`mcp__claude_ai_Sefaria__get_text`). זה המאמת היחיד. ציטוטים שלא ניתן לאמת מסומנים `[UNVERIFIED]`.

## התאמה לסוגות

### פילוסופיה וטקסטים אקדמיים

דגש על דיוק מושגי ולוגיקה. `linguistic-editor` משתמש ב־skill `connectives` כדי לוודא שימוש נכון במילות קישור — ברם, אולם, שכן, הואיל ו, אי לכך, לפיכך — בהתאם לסוג הקשר הלוגי.

### אוטוביוגרפיה וסיפורי חיים

שיפור הנרטיב תוך שמירה על אותנטיות. `literary-editor` בודק עיצוב דמויות, ציר זמן עקבי, ובניית עולם.

### טקסטים דתיים וספרי קודש

ה־skill `cite-master` כולל מסלול ייעודי לציטוט נכון של תנ"ך, בבלי, ירושלמי ומדרש, ומאמת כל אזכור מול Sefaria דרך MCP. אין להשמיט או להוסיף מהטקסט המקורי ללא סימון בסוגריים מרובעים `[...]`. עבור שירה או טקסטים דתיים — `niqqud-pass` מפעיל סבב הגהה לניקוד.

## ה־Review Skill: גיוון, מקצב, וביטול סימני AI

האתגר הגדול בכתיבה עם בינה מלאכותית בעברית הוא המראה ה"רובוטי" של הטקסט. ה־skill `review-style` נועד לפתור זאת:

1. **גיוון מבנה המשפטים (Burstiness)** — מודל AI נוטה לאורך משפטים בינוני קבוע. הכישור מציע שילוב של משפטים קצרים וחדים לצד משפטים ארוכים — מקצב אנושי.
2. **הסרת ביטויים שחוקים** — מחיקת פתיחים גנריים כמו "בעולם המשתנה של היום" או "חשוב לזכור ש...".
3. **שיפור המנגינה הלשונית** — בדיקה של זרימת הטקסט והתאמת המשלב.

|---|---|---|

## עימוד וטיפוגרפיה — סטנדרטים ישראליים

`typesetting-agent` עובד לפי:

- **גופן:** Frank Ruhl Libre Bold/Book — הקריא ביותר בדפוס עברי.
- **שוליים:** מונעים בליעת טקסט בקפל המרכזי.
- **כותרות רצות:** שם הספר באי־זוגי, שם הפרק בזוגי.
- **חלוקה לשערים:** התחלת פרק חדש בעמוד שמאל (זוגי) לפי המסורת המקצועית.

ראה [`skills/hebrew-typography/references/`](./plugins/hebrew-book-producer/skills/hebrew-typography/references/) לפרטים מלאים.

## גיליון דפוס
