# מדריך למשתמש: hebrew-book-producer

> **למי המדריך הזה?** למחברים שכותבים ספר עיון, אוטוביוגרפיה, ספר פילוסופיה או ספר דתי בעברית — ואין להם רקע טכני. אין צורך לדעת לכתוב קוד.

המדריך הזה מסביר במילים פשוטות מה הפלאגין עושה, איך מתקינים, ואיך עובדים איתו על הספר שלכם.

---

## מה זה הפלאגין הזה, ולמה אני צריך אותו?

תחשבו על מה שקורה כשמוציאים ספר בהוצאה לאור מקצועית בישראל:

1. הסופר מגיש כתב יד.
2. **לקטור** קורא אותו ומחליט אם הוא בשל לפרסום.
3. **עורך ספרותי** עובד על המבנה — סדר הפרקים, הקצב, הדמויות או הטיעונים.
4. **עורך לשוני** עובר משפט אחר משפט ומשפר את העברית.
5. **מגיה** קורא הכל פעמיים — לפני העימוד ואחריו — ותופס שגיאות.
6. **גרפיקאי / מעמד** מעצב את הספר לדפוס (פונט, שוליים, כותרות רצות).

הפלאגין מבצע את כל הצוות הזה אוטומטית. במחשב שלכם. תוך שהוא **שומר על הקול שלכם** ולא הופך את הספר ל-"AI גנרי".

המטרה: לחסוך זמן וכסף, ולהוציא ספר באיכות הוצאה לאור — אבל בלי להחליף את הסופר.

---

## מה הפלאגין **לא** עושה

- ❌ הוא לא כותב את הספר במקומכם. הוא עורך ומלטש מה שאתם כתבתם.
- ❌ הוא לא ידפיס את הספר. הוא ייצר מפרט מקצועי שגרפיקאי או InDesign יכול לעבוד איתו.
- ❌ הוא לא שולח את הספר להוצאה. ההפצה היא תפקיד שלכם.
- ❌ הוא לא מחליף את ההחלטות שלכם. הוא מציע — אתם מאשרים.

## דרישות מקדימות

לפני שתתחילו, צריך:

1. **Claude Code מותקן** — אפליקציית Claude לטרמינל. אם אין לכם — בקרו ב־[claude.ai/code](https://claude.ai/code).
2. **כתב יד טיוטה ראשונה** של הספר, שמור בקובץ אחד או יותר בפורמט Markdown (`.md`). אם הקובץ שלכם ב-Word — שמרו אותו כ-`.txt` ושנו את הסיומת ל-`.md`.
3. (אופציונלי, מומלץ) **CandleKeep CLI** — מערכת ידע אישית שמאפשרת לפלאגין להישען על מדריך הכתיבה הקנוני שיצרתם. ראו [candlekeep.cloud](https://candlekeep.cloud).

הפלאגין יעבוד גם בלי CandleKeep — פשוט במצב מצומצם.

## התקנה (פעם אחת בלבד)

פתחו טרמינל. הקלידו:

```bash
cd ~/dev/hebrew-book-producer
claude plugin install --local .
```

או, אם אתם רוצים שהפלאגין יהיה זמין בכל פרויקט:

```bash
ln -s ~/dev/hebrew-book-producer ~/.claude/plugins/local/hebrew-book-producer
```

זהו. הפלאגין מותקן.

## עבודה עם הפלאגין: מדריך צעד-אחר-צעד

### שלב 1: צרו תיקייה לספר שלכם

פתחו טרמינל ועברו לתיקייה שתהיה הבית של הספר. למשל:

```bash
mkdir ~/Documents/הספר-שלי
cd ~/Documents/הספר-שלי
```

העתיקו את כתב היד (קבצי `.md`) לתיקייה הזו.

### שלב 2: הפעילו את Claude Code

```bash
claude
```

תראו שורת פקודה אינטראקטיבית.

### שלב 3: התחילו פרויקט חדש

הקלידו:

```
/init
```

הפלאגין ישאל אתכם כמה שאלות בעברית:

- **שם הספר?**
- **שם המחבר?**
- **סוגה?** (פילוסופיה / אוטוביוגרפיה / דתי / מדע פופולרי)
- **כמה מילים אתם מכוונים?** (טווח טיפוסי: 40,000–80,000)
- **סגנון ציטוטים?** (Chicago / APA / חז"ל)
- **ניקוד?** (כן רק לשירה או טקסטים דתיים)
- **תאריך יעד?**

הפלאגין ייצר שני קבצים חשובים:

- **`book.yaml`** — מטא-דאטה של הספר.
- **`AUTHOR_VOICE.md`** — שלד של "טביעת הקול" שלכם. **זה הקובץ הכי חשוב בפרויקט.** מלאו אותו עכשיו (ראו השלב הבא).

### שלב 4: מלאו את `AUTHOR_VOICE.md` (חמש דקות שיחסכו לכם שעות)

פתחו את הקובץ `AUTHOR_VOICE.md` בעורך טקסט (TextEdit, Notepad, VSCode — מה שיש לכם). מלאו אותו לפי השלד:

- **פרסונה** — פסקה אחת: מי המספר? איזה טון?
- **משלב** — האם אתם כותבים בעברית ספרותית גבוהה? עברית מודרנית? מעורבת?
- **ביטויים מועדפים** — מילים שאתם אוהבים להשתמש בהן.
- **ביטויים אסורים** — מילים שאתם **לא** רוצים שהפלאגין ישתמש בהן (למשל "בעולם המשתנה של היום").
- **חמש פסקאות לדוגמה** מהכתיבה הקודמת שלכם — שיהיו ה"מצפן" של הפלאגין.

ככל שתשקיעו יותר בקובץ הזה, כך הפלאגין ידייק יותר בעריכה ולא יהפוך את הספר לגנרי.

### שלב 5: הערכת לקטור

הריצו:

```
/lector manuscript.md
```

(החליפו את `manuscript.md` בשם הקובץ שלכם.)

הפלאגין יקרא את כל כתב היד וייצר קובץ בשם `LECTOR_REPORT.md` שכולל:

- **תקציר** של מה הספר עושה.
- **מיצוב שוק** — מי הקהל, ספרים דומים בישראל.
- **ניתוח מבני** — איפה החוזק, איפה החולשה.
- **המלצה** — *Go* (להמשיך), *Go with major revisions* (לתקן ואז להמשיך), או *No-go* (יש בעיה יסודית).

קראו את הדוח לפני שתמשיכו. **אם הוא אומר שצריך לתקן משהו יסודי — תקנו לפני שלב הבא.**

### שלב 6: עריכה (ספרותית + לשונית)

```
/edit
```

הפלאגין יריץ שני סוכנים ברצף:

1. **עורך ספרותי** — עובר על המבנה: סדר פרקים, קצב, נושאים. אולי יחתוך פרקים שלמים. תמיד יסביר למה.
2. **עורך לשוני** — עובר משפט אחר משפט: תחביר, משלב, מילות קישור (ברם / לפיכך / שכן), הסרת ביטויי AI שחוקים.

כל שינוי נשמר אוטומטית בקובץ `.book-producer/memory.md` — ככה אם תתקנו אותו ידנית בעתיד, הוא יזכור.

### שלב 7: הגהה (פעם ראשונה)

```
/proof
```

הפלאגין יקרא הכל בארבע רמות (אות, מילה, משפט, רעיון). יתקן שגיאות כתיב, פיסוק, האחדת כתיב.

### שלב 8: עימוד (מפרט בלבד)

```
/typeset
```

הפלאגין יייצר `TYPESETTING_BRIEF.md` — מסמך מקצועי שכולל:

- גופן: Frank Ruhl Libre Book (הקריא ביותר בדפוס עברי).
- גודל אות, רווחים, שוליים, כותרות רצות.
- כללי שערים, פרקים, ציטוטים.

**הפלאגין לא יוצר PDF.** הוא יוצר **מפרט** שאתם או גרפיקאי לוקחים ל-InDesign / LaTeX.

### שלב 9: הגהה (פעם שנייה — אחרי העימוד)

אחרי שעיצבתם את הספר וייצרתם PDF טיוטה, הריצו:

```
/proof
```

הפלאגין מזהה אוטומטית שהוא בסבב השני. הוא יחפש שגיאות שנגרמו מהעימוד עצמו (גרשיים שבורים, מספור שגוי, שורות יתומות).

### שלב 10: סיימתם!

הספר מוכן לדפוס.

## מה לעשות אם משהו לא עובד

### "הפלאגין לא מצא את הספר"
ודאו שהקובץ קיים בתיקייה הנוכחית והוא בפורמט `.md`.

### "הפלאגין מציע שינויים שאני לא אוהב"
זה נורמלי. אתם **תמיד** יכולים לדחות. שני דרכים:

1. תגידו "לא, תחזור אחורה" בצ'אט — הפלאגין יחזיר.
2. תקנו ידנית — הפלאגין ילמד מהתיקון שלכם.

אחרי 3 דחיות של אותו סוג, הריצו:

```
/voice
```

הפלאגין יציע להוסיף את הסגנון שדחיתם ל-`AUTHOR_VOICE.md` באופן קבוע.

### "הפלאגין נשמע 'AI'"
זה הסימן ש-`AUTHOR_VOICE.md` חסר תוכן. חזרו לשלב 4 ומלאו אותו ביסודיות. שלוש פסקאות אמיתיות שלכם שווה יותר מאלף הוראות.

### "אני רוצה לתקן רק פרק אחד"
כל פקודה מקבלת ID של פרק:

```
/edit ch04
/proof ch04
```

### "איך אני מבטל את הכל ומתחיל מחדש?"
מחקו את התיקייה `.book-producer/`. הספר עצמו יישאר בלי שינוי. הפלאגין ישאל אם להתחיל מחדש בריצה הבאה.

## שאלות נפוצות

**ש: כמה זמן זה לוקח?**
ת: לספר של 60,000 מילים — האוטומציה רצה כ-1–2 שעות שעות-מחשב לכל סבב מלא. אבל אתם תקראו ותתקנו בין הסבבים, אז התהליך האמיתי הוא לרוב 2–4 שבועות.

**ש: זה יחליף עורך אנושי?**
ת: לא. הוא משלים. עורך אנושי מקצועי יביא ניואנסים שאף AI עדיין לא יכול. לכן ההמלצה: השתמשו בפלאגין לעבודה הראשונית, אז השליחו לעורך אנושי לליטוש סופי.

**ש: השיר/הספר הדתי שלי דורש ניקוד. הפלאגין מתמודד?**
ת: כן. הפעילו `niqqud: true` בקובץ `book.yaml`. הפלאגין יריץ סבב הגהה ייעודי לניקוד.

**ש: יש לי טקסט עם ציטוטים מהתנ"ך / תלמוד. הפלאגין יודע לטפל?**
ת: כן. בחרו `genre: religious` (או כל סוגה אחרת — אימות הציטוטים פועל בכל מצב). הפלאגין מאמת **כל** ציטוט מחז"ל ישירות מול **Sefaria** (האתר הקנוני). אם מציאת הציטוט נכשלת — הוא מסומן `[UNVERIFIED]` בכתב היד כדי שתוכלו לבדוק ידנית. אין צורך בהתקנה של ספריה נוספת — הפלאגין משתמש ב-MCP של Sefaria (אם זמין ב-Claude Code) או פונה ל-API הציבורי של Sefaria דרך הסקריפט המצורף.

**ש: איפה נשמרות המחשבות שלי על הספר — רעיונות, נושאים, החלטות עיצוב?**
ת: ב-CandleKeep, ב"מחברת התזה" של הספר. המחברת מוגדרת בשדה `thesis_notebook:` בקובץ `book.yaml` ומכילה את ID הפריט ב-CandleKeep. הוסיפו רעיונות עם `/thesis append "<טקסט>"`, צפו עם `/thesis show`. הסוכנים יקראו את המחברת בתחילת כל סשן.

**ש: האם הפרטיות שלי בטוחה?**
ת: כן. הפלאגין רץ מקומית במחשב שלכם. אין שליחה לשרת חיצוני מלבד מודל ה-AI עצמו (Claude). כתב היד לא נשמר בענן.

**ש: אני רוצה לכתוב באנגלית, לא בעברית. אפשר?**
ת: לא. הפלאגין הזה ייעודי לעברית. לאנגלית יש פלאגינים אחרים.
